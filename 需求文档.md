我想开发一个“个人AI工作台”全栈项目，请帮我设计并实现完整系统（后端 + 前端），可以自由发挥具体实现，但必须满足以下核心功能和目标。

====================
一、产品目标
====================

这是一个我个人每天会使用的AI工具，核心作用是：

- 记录每天做了什么
- 管理自己的任务
- 帮我拆解任务（可以AI拆解，也可以手动修改）
- 自动总结工作内容（手动点击生成总结）
- 帮我做周期复盘（周报 / 月报，每周、每月、年中、年终自动生成）

重点不是复杂功能，而是：
👉 简单、好用、能每天用

====================
二、核心功能（必须实现）
====================

1. 任务管理
- 创建任务
- 查看任务列表
- 标记完成

2. AI任务拆分
- 输入一个目标或任务
- AI自动拆分为多个子任务

3. 工作记录
- 输入每日工作内容（自由文本）
- 查看历史记录

4. AI自动总结
- 每条记录自动生成结构化总结

5. 周报 / 月报
- 基于历史记录自动生成总结

6. 基础统计
- 总记录数
- 已完成任务数
- 活跃天数

====================
三、关键设计要求
====================

- AI必须参与核心流程（不是简单聊天工具）
- 尽量减少用户操作（自动总结、自动记录）
- 流程闭环：
  任务 → 执行 → 记录 → 总结

====================
四、技术要求
====================

后端：
- Python + FastAPI
- SQLite数据库，考虑后续要切换mySQL或PostgreSQL
- 模块清晰（tasks / records / summary / ai_service）

前端：
- React + Vite
- 使用 Ant Design 组件库（antd）
- 使用函数组件 + Hooks

AI：
- 封装统一调用模块（先用mock也可以）
- 后续可扩展真实API

====================
七、设计原则
====================

- 优先“能用”
- 优先“简单”
- 优先“真实使用场景”

====================
最终目标：
====================

生成一个我可以每天使用的个人AI工作台，后续还要扩展其他功能模块，通过不同菜单区分
界面简洁，功能实用，AI能真实参与我的工作流程。
